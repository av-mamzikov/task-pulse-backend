# Краткое описание функциональности разрабатываемого MVP:

Веб-сервис **"TaskPulse"** — минимальный менеджер задач с трекером статусов.
Пользователь должен иметь возможность:
1. Создавать задачу (название, описание, приоритет, срок).
2. Просматривать список всех задач.
3. Изменять статус задачи: `Новое`, `В работе`, `Готово`.
4. Удалять задачу.
5. Фильтровать список по статусу и приоритету.
6. (опционально) Добавлять заметку или комментарий к задаче.

Требования:
* Данные хранятся в PostgreSql.
* Интерфейс должен быть минималистичным, но понятным.
* После перезапуска приложения данные должны сохраняться.

- Предпочтительный стек:
  - Backend: NodeJS (express)
  - Frontend: React + TypeScript + Vite
  - БД: выбери самый подходящий вариант
  - Хостинг/запуск: Отдельные репозитории для бэкенда и фронтенда, каждый со своим Dockerfile
